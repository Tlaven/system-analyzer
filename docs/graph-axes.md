# Graph 呈现调整空间

> 版本：v1.0
> 日期：2026-06-13

---

## 核心思想

同一个 graph 数据，不改变可视化形式（不是邻接矩阵、桑基图等），只调整**呈现的粒度和方式**。用户在每个维度上独立选择，从极简到全量是一个连续光谱。

样式改外观，模式改行为，信息层级改颗粒度——这些都不触及"视图"层级。

### 1.1 形状

| 选项 | 说明 |

| 圆角矩形 | 当前默认 |
| 圆形 | arc 绘制 |
| 胶囊 | 两端半圆 |
| 菱形 | 旋转 45° 正方形 |
| 六边形 | 六角形 |

### 1.2 尺寸

| 选项 | 说明 |
|------|------|
| 固定尺寸 | 所有节点统一大小 |
| 按标签自适应 | 当前实现，有最大宽度限制 |
| 按属性值缩放 | 节点大小映射某个数值属性 |
| 按连接数缩放 | 连接越多的节点越大 |

### 1.3 信息密度

这是最核心的维度——节点上直接展示多少信息。

| 层级 | 显示内容 |
|------|---------|
| 极简 | 仅名称（当前默认） |
| 轻量 | 名称 + 分组标签 |
| 中等 | 名称 + 关键属性值 |
| 完整 | 名称 + 描述摘要 + 所有属性 + 端口列表 |

实现上：中等和完整模式下节点高度动态扩展，类似卡片式布局。属性以 key: value 行内显示。

### 1.4 色彩编码

| 按属性值 | 数值属性 → 渐变色映射 |
| 按连接数 | 连接数 → 颜色深浅 |

### 1.5 边框

## 四、布局的调整维度

### 4.1 位置模式

| 选项 | 说明 |
|------|------|
| 手动固定 | 节点有坐标，拖拽是移动（当前） |
| 弹性物理 | 力导向持续运行，拖拽是施加临时力 |
| 算法一次性 | 力导向/圆形/层次，计算完固定 |

弹性物理模式和手动模式的区别：
- 手动：节点有固定 (x, y)，拖拽改变坐标
- 弹性：节点坐标每帧被力场重新计算，拖拽相当于临时固定节点位置，松开后恢复物理模拟

### 4.2 方向约束

| 选项 | 说明 |
|------|------|
| 自由 | 节点可在任意方向 |
| 从上到下 | 层次布局强制纵向 |
| 从左到右 | 层次布局强制横向 |
| 径向 | 从中心节点向外辐射 |

### 4.3 对齐辅助

| 选项 | 说明 |
|------|------|
| 自由 | 当前默认 |
| 吸附网格 | 节点松手时对齐最近网格点 |
| 吸附节点 | 拖拽时靠近其他节点自动对齐 |

## 七、全局信息层级

### 7.1 信息缩放滑块

一个全局控制，同时调整节点和边的信息密度：

```
极简 ◄──────────────────────────────────────► 全量

节点：仅名称                    → 名称+属性 → 完整卡片
边：  仅线条                    → 带标签    → 带全部描述和数值
端口：隐藏                      → 显示      → 带标签
悬停：无                        → tooltip   → 高亮关联
```

这是一个快捷方式，同时移动多个轴。用户也可以单独调整某个轴覆盖全局设置。

## 九、设计原则

1. **每个轴独立**：用户可以在任何轴上单独调整，不影响其他轴
2. **预设打包**：大部分用户不需要逐轴调整，提供几套预设组合（如"简洁"、"详细"、"演示"）
3. **持久化**：所有选择保存到 localStorage，下次打开保持
4. **向后兼容**：新维度都有默认值，旧数据无需迁移
5. **渐进暴露**：新手看到预设就够用，高级用户可以展开逐轴调整
